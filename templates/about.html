About


What is this?

Tumorclass.info is a web application that allows users to upload MRI images of tumors and classify them into NORMAL (no tumor found), LGG (low grade glioma), and HGG (high grade glioma).

What data is used?

The dataset used for training the model is the The REMBRANDT (REpository for Molecular BRAin Neoplasia DaTa) Dataset from The Cancer Imaging Archive (TCIA). The data set used is a smaller subset of the original manually labeled and provided by Dr. Saed Khawaldeh.
Note! Some augmentation (primarily rotation and flipping) is done on the dataset due to its relatively small size.

How is the classification model implemented?

The Convoluted Neural Networks (CNN) architectures are implemented in TensorFlow/Keras. Some by loading model architectures and imagenet weights from tensorflow, others by building the model from scratch.

How is the model trained?

All the models share the same preprocessing and training pipeline. The pipeline is lets the user easily fine-tune the hyperparameters, change optimizer, number of epochs, callbacks, etc.

How is the this website implemented?

The website is built using Python, FastAPI, and Jinja2 templating engine.

What happens to uploaded images?

The images are uploaded to the server and then processed by the model. Since underlying functions don't handle in-memory images, the images are stored on the server. Everytime the server restarts all uploaded images are deleted. Other than that, there are no other automated processes that delete images. But by clicking the "Delete" button, the user can delete the image.

How can I contribute?

If you have access to a dataset manually annotated (NORMAL, LGG, or HGG) by radiologist, neurologists, or other applicable professionals please contact me at lewi@uberg.me

What is it for?

It is part of the senior year project for Applied Data Science at Noroff University College.
